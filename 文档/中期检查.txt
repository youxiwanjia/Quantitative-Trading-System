  二、潜在 Bug（严重程度：高）

  1. Baostock 登录状态管理问题

  文件: fetcher_daily.py:26-29, fetcher_minute.py:25-28, fetcher_index.py:35-38

  def _ensure_login(self):
      if not self.logged_in:
          bs.login()  # 未检查返回值
          self.logged_in = True

  问题：登录失败时未抛出异常，后续操作会静默失败。对比 fetcher_minute_base.py:30-34 有正确处理。

  建议：统一检查登录返回值。

  三、性能问题（严重程度：中）

  1. 日线数据加载效率低

  文件: data/loader.py:100-120

  for code in stocks:
      for year in years:
          file_path = ...
          df = pd.read_parquet(file_path)  # 逐文件读取

  问题：当股票数量大时，会产生大量 I/O 操作。3000只股票 × 多年 = 数千次文件读取。

  建议：考虑批量读取或使用 Parquet Dataset API。

  ---
  2. 因子计算无缓存

  文件: factors/engine.py

  问题：每次调用 compute_factor 都重新计算，无结果缓存。多策略回测时会重复计算相同因子。

  文件: data/trading_calendar.py:137

  return d in set(self._df['date'])  # 每次都创建新 set

  建议：将 set 缓存为实例变量。

  六、最佳实践改进建议

  1. 添加连接池管理

  # 建议创建 Baostock 连接管理器
  class BaostockConnectionManager:
      _instance = None
      _connected = False

      @classmethod
      def get_connection(cls):
          if not cls._connected:
              result = bs.login()
              if result.error_code != '0':
                  raise ConnectionError(result.error_msg)
              cls._connected = True
          return bs

  2. 添加数据验证层

  def validate_stock_code(code: str) -> bool:
      """验证股票代码格式"""
      pattern = r'^(sh|sz)\.\d{6}$'
      return bool(re.match(pattern, code))
